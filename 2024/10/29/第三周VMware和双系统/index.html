VMware下载及安装

下载安装包

  • 因为官网下载速度比较慢，所以选择去GitHub社区进行VMware的下载（201853910/VMwareWorkstation）
  • 如图，图片上已经附上下载链接和注册码
  • GitHub有proxy代理，下载速度飞起。

安装VMware

  • 右键以管理员身份运行镜像文件
  • 换安装位置
  • 取消勾选更新和用户体验计划
  • 安装，输入密钥。我用的密钥是MC60H-DWHD5-H80U9-6V85M-8280D
  • 安装完成。

Ubuntu下载及安装

下载安装包

  • 使用中科大的镜像网址下载24.04的Ubuntu
  • http://mirrors.ustc.edu.cn/ubuntu-releases/
  • 下载的是桌面版的

安装Ubuntu到虚拟机中

  • 创建虚拟机，点击自定义高级。
  • 稍后安装系统，因为想先把虚拟机配置清楚
  • 预建立操作系统。
  • 换文件位置。
  • 选择处理器数量。
  • 一直点击“下一步”，按照安装向导的默认选项选择，直到完成。
  • 移除打印机- 去除sata 1：0警告，在sata选项中选择iso镜像
  • 确定后开启虚拟机等待系统安装。
  • 等待进入系统

使用VI编辑器编写c语言程序

安装vim编辑器

  • 使用Ctrl+alt+T打开命令系统终端，输入命令更新软件包列表sudo apt-get update
  • 完成更新后输入命令安装vim编辑器sudo apt-get install vim
  • 安装完成后创建文件夹mkdir test00
  • 输入命令检查是否创建成功 ls

编辑c语言程序

  • 创建一个新文件 vim helloworld.c
  • 点击小写i进入编辑模式
  • 编写hello world
  • 点esc退出输入；进入命令行模式
  • Wq代表保存退出q！表示不保存退出
  • 编辑好后退出

使用gcc编译器编译

下载gcc编译器

  • 输入命令下载gcc编译器 sudo apt-get install gcc

编译运行输出结果

  • 编译格式：gcc 源文件名 -o 生产执行的文件名
  • 输入命令进行编译gcc helloworld.c -o helloworld
  • 输出结果./helloworld

安装双系统

下载安装镜像工具

  • 下载win32diskimager

制作Ubuntu安装盘

  • 选择镜像文件iso写入u盘

Gpt安装

问题及解决

问题1：VMware安装过程中无法启动虚拟机

  • 描述：在安装VMware后，尝试启动虚拟机时出现错误，无法进入Ubuntu系统。
  • 解决方案：检查虚拟机配置，确保CPU、内存等硬件资源分配充足。同时，检查Ubuntu安装镜像文件是否正确。如果问题仍然存在，尝试重新安装Ubuntu或更新VMware软件。

问题2：VI编辑器无法正常工作

  • 描述：在尝试使用VI编辑器编写C语言程序时，发现无法正常输入或保存文件。
  • 解决方案：确认VI编辑器是否正确安装。如果已安装，可能是环境变量设置问题。尝试通过终端输入vimtutor来查看VI编辑

问题3：无法使用gcc编译器编译C语言程序

  • 描述：在尝试使用gcc编译器编译C语言程序时，系统提示找不到gcc命令。
  • 解决方案：确保gcc编译器已经正确安装。如果未安装，可以通过命令sudo apt-get install gcc来安装。安装完成后，检查gcc版本以确认安装成功，使用命令gcc --version。

问题4：GRUB引导加载器错误

  • 描述：在安装Ubuntu后，重启计算机时出现GRUB引导加载器错误，无法正常启动。
  • 解决方案：如果GRUB未正确安装，可以通过Ubuntu的安装盘进入救援模式，然后使用sudo update-grub命令来更新GRUB配置。

问题5：Ubuntu安装过程中网络连接不稳定

  • 描述：在Ubuntu安装过程中，网络连接断断续续，导致安装进度缓慢或失败。
  • 解决方案：在安装过程中，尝试连接到有线网络或切换到稳定的无线网络。如果网络连接问题仍然存在，可以考虑在安装过程中暂时禁用网络连接，安装完成后再进行网络配置。
