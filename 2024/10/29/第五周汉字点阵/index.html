Windows下编码问题

  • 描述
      • 描述
    • 一打开文件发现乱码，一看是编码格式冲突的问题
    • 因为此源文件默认是gbk编码，而我设置的是默认utf8编码，所以会出现中文乱码

Linux下编码问题2

  • 描述
    • 因为我需要在Linux编写c文件但是不改变系统编码utf-8所以决定使用libiconv包的iconv命令解决Linux和Windows间文件乱码的问题
    • 不改变系统编码的原因是再locale写入lang=zn-cn gbk打不开终端
  • 解决方案
    • 下载
      • wget http://ftp.gnu.org/pub/gnu/libiconv/libiconv-1.13.1.tar.gz
    • 解压
      • tar -xzvf libiconv-1.13.1.tar.gz
      • cd libiconv-1.13.1
    • configure配置
      • sudo make -j8
      • sudo make install
    • 创建软连接
      • sudo ln -s /usr/local/lib/libiconv.so /usr/lib
      • sudo ln -s /usr/local/lib/libiconv.so.2 /usr/lib/libiconv.so.2
    • 使用gcc编译目标文件
      • gcc -o q q.c -L/usr/lib -liconv
